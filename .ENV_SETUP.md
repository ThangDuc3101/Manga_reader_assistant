# ЁЯУЭ Environment Setup Guide

Quick reference for setting up your `.env` file and API key.

---

## ЁЯЪА Quick Setup (2 minutes)

### Step 1: Copy Template
```bash
cp .env.example .env
```

### Step 2: Get Roboflow API Key
1. Go to https://roboflow.com
2. Sign up (free) or log in
3. Click **Settings** тЖТ **Roboflow API**
4. Copy your **Private API Key**

### Step 3: Edit .env File

**On Linux/Mac:**
```bash
nano .env
```

**On Windows:**
```bash
notepad .env
```

Find this line:
```
ROBOFLOW_API_KEY=[REDACTED:api-key]
```

Replace with your actual key:
```
ROBOFLOW_API_KEY=abc123xyz...
```

Save the file.

### Step 4: Run App
```bash
streamlit run main.py
```

Done! ЁЯОЙ

---

## ЁЯУЛ What Each Variable Does

### Required
- **ROBOFLOW_API_KEY**: Your private API key from Roboflow
  - Get it at: https://roboflow.com/settings/api
  - Keep it secret! Don't share or commit to git

### Optional
- **ROBOFLOW_MODEL**: Model name (default: "manga-bubble-detect")
- **ROBOFLOW_VERSION**: Model version number (default: 1)

## ЁЯФТ Security Tips

тЬЕ **DO:**
- Store API key in `.env` file
- Add `.env` to `.gitignore` (already done)
- Keep key private and secure
- Regenerate key if compromised

тЭМ **DON'T:**
- Commit `.env` to git
- Share API key with others
- Use in public code or GitHub
- Hardcode key in Python files

## ЁЯФД What If I Don't Set Up .env?

The app will still work! It has two fallback options:

1. **Environment Variable**: `export ROBOFLOW_API_KEY="your_key"`
2. **YOLOv8s Model**: Auto-downloads, no setup needed
   - In config.py: `USE_ROBOFLOW = False`

## ЁЯЫая╕П Troubleshooting

### Error: "ROBOFLOW_API_KEY not set"
- Make sure `.env` file exists: `ls -la .env`
- Check API key is correct
- Verify file format: `ROBOFLOW_API_KEY=your_key` (no spaces)

### Error: "API key invalid"
- Log in to Roboflow and check key hasn't expired
- Generate new key if needed
- Update `.env` with new key

### Module error: "python-dotenv not found"
```bash
pip install python-dotenv
# Or reinstall all dependencies:
pip install -r requirements.txt
```

### .env file not being read
- Make sure file is in project root: `/Manga_reader_assistant/.env`
- Check file has correct name (not `.env.txt` or `env`)
- Restart your app after creating/editing `.env`

## ЁЯТб Pro Tips

### Use different keys for dev/prod
```bash
# Development .env
ROBOFLOW_API_KEY=dev_key_xxx

# Production (different key)
ROBOFLOW_API_KEY=prod_key_yyy
```

### Enable debug logging
```bash
# In .env
# ROBOFLOW_DEBUG=1  # (if using custom code)

# Or in config.py
DEBUG_MODE = True
LOG_LEVEL = "DEBUG"
```

### Test without API key
```bash
# Temporarily disable Roboflow
USE_ROBOFLOW=False
# App will use YOLOv8s fallback
```
